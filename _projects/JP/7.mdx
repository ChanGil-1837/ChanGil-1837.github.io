# 0で割る

数学の授業で先生がいつも強調していたルールがあります。それは、どんな数も0で割ることはできないということです。1割る0、10割る0、100割る0…これらの計算はすべて定義されていません。<br/>

では、コンピュータにこの計算をさせたらどうなるでしょうか？コンピュータは数学の基本ルールにそのまま従うのでしょうか？  
一度調べてみることにしましょう。<br/>

<br/>

## 数学で「0で割る」が不可能な理由

コンピュータがどのような計算をするのかを理解する前に、まず数学で0で割ることがなぜ不可能なのかをもう一度確認しておくのが良いでしょう。<br/>

割り算は掛け算の逆演算です。例えば、6割る3は2です。これは3×2=6だからです。  
では、6÷0を計算してみましょう。0×?=6となる数を見つけなければなりませんが、どんな数に0を掛けても結果は常に0です。したがって、6を作ることができる数はこの世に存在しません。<br/>

もし0÷0を計算したらどうなるでしょうか？0×?=0となる数を見つけなければなりません。ところが、どんな数に0を掛けても0になるので、答えは1にもなりうるし、2にもなりうるし、100にもなりえます。  
答えが一つに定まらないため、これも数学では定義しません。<br/>

<br/>

## それでもやらせてみたら？

**整数型の0除算**を試みると、ほとんどの言語でエラーが発生します。  
C#では**DivideByZeroException**のようなランタイムエラーが発生します。<br/>

```csharp
int a = 10;
int b = 0;
Console.WriteLine(a / b); // DivideByZeroExceptionが発生します
```
```pgsql
chanhogil@chanhos-MacBook-Pro ConsoleApp1 % dotnet run
Unhandled exception. System.DivideByZeroException: Attempted to divide by zero.
at HelloWorld.Program.Main(String[] args) in
```

**実数型**では少し動作が異なります。<br/>
1.0を0.0で割ると**Infinity**として処理されます。
```csharp
float a = 1.0f;
float b = 0;
Console.WriteLine(a / b);
```
```pgsql
chanhogil@chanhos-MacBook-Pro ConsoleApp1 % dotnet run
∞
```
0.0を0.0で割ると**NaN (Not a Number)**として処理されます。
```pgsql
chanhogil@chanhos-MacBook-Pro ConsoleApp1 % dotnet run
NaN
```

整数型には無限大やNaNの概念がないため、無条件に例外が発生します。<br/>
<br/>
**結果**
<br/>
整数型の0除算はC#で**DivideByZeroException**が発生し、プログラムが停止します。<br/>
実数型では**Infinity**や**NaN**として処理される可能性がありますが、整数型にはそのような概念がないためエラーになります。<br/>
<br/>
**結論**
>数学的に不可能な演算であるため、整数型の0除算は常に例外が発生します。<br/>
>実数型は**Infinity**や**NaN**として処理可能です。<br/>
>小数点計算時には、可能であれば整数に変換して計算する方が安全です。<br/>
